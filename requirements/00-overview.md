# 프로젝트 개요

## 서비스명
Pole - 개발팀 인트라넷

## 서비스 목적
20명 규모 개발팀이 Figma 디자인을 중심으로 협업하고, 프로젝트를 관리하며, 팀 내부 소통을 원활하게 하기 위한 인트라넷 플랫폼

## 대상 사용자
- 프론트엔드 개발자
- 백엔드 개발자
- UI/UX 디자이너
- PM / PO
- QA 엔지니어
- 팀 리드 / 관리자

## 기술 스택
- **프레임워크**: Next.js 16 (App Router)
- **UI 라이브러리**: Mantine UI
- **언어**: TypeScript
- **스타일링**: Tailwind CSS + Mantine
- **외부 연동**: Figma MCP Server

## 핵심 모듈 (5단계)

| Phase | 모듈 | 설명 |
|-------|------|------|
| Phase 1 | 기반 구축 | 레이아웃, 인증, 팀원 프로필, 조직도 |
| Phase 2 | Figma 허브 | Figma Explorer 고도화, 디자인 리뷰, 핸드오프 |
| Phase 3 | 프로젝트 관리 | 프로젝트 보드, 업무 할당, 타임라인 |
| Phase 4 | 팀 커뮤니케이션 | 공지사항, 팀 게시판, 회의록, 사내 위키 |
| Phase 5 | 개발 도구 & 대시보드 | 배포 현황, 코드 스니펫 공유, 팀 대시보드 |

## 구현 원칙
1. Figma 연동이 서비스의 핵심 차별점 — 모든 모듈에서 Figma 연결 고려
2. 20명 팀에 적합한 경량 설계 — 과도한 기능보다 실용성 우선
3. 각 Phase 완료 시 독립적으로 사용 가능한 상태 유지
