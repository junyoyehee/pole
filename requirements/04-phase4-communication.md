# Phase 4: 팀 커뮤니케이션

## 목표
팀 내부의 공지사항, 자유 소통, 회의록, 지식 공유를 위한 커뮤니케이션 도구를 제공합니다.

---

## 4.1 공지사항

### 설명
팀 리드/관리자가 중요 사항을 전달하는 공간

### 기능
- 공지 작성 (LEAD, ADMIN만)
- 공지 목록 (최신순, 고정 공지 상단)
- 리치 텍스트 에디터
- 파일 첨부
- 읽음 확인 (누가 읽었는지 표시)
- 중요 공지 고정 (핀)
- 대시보드에 최근 공지 위젯

---

## 4.2 팀 게시판

### 설명
팀원 누구나 자유롭게 글을 작성하는 소통 공간

### 카테고리
| 카테고리 | 설명 |
|---------|------|
| 자유 | 자유로운 대화 |
| 기술 공유 | 기술 아티클, TIL, 발견한 도구 등 |
| 질문 | 개발/디자인 관련 질문 |
| 건의 | 팀 운영 개선 건의 |

### 기능
- 게시글 CRUD
- 마크다운 에디터
- 댓글 / 대댓글
- 좋아요
- 검색 (제목, 내용)
- 카테고리 필터

---

## 4.3 회의록

### 설명
팀 회의 내용을 기록하고 공유하는 공간

### 회의록 작성
| 항목 | 설명 |
|------|------|
| 회의 제목 | 예: "주간 스프린트 회의" |
| 회의 일시 | 날짜 + 시간 |
| 참석자 | 팀원 선택 (복수) |
| 회의 유형 | 정기회의 / 스프린트 / 기획 / 회고 / 기타 |
| 안건 | 논의 항목 목록 |
| 회의 내용 | 마크다운 에디터 |
| 결정 사항 | 회의에서 결정된 내용 (하이라이트) |
| 액션 아이템 | 후속 조치 항목 (담당자 + 기한) |
| 관련 Figma | 회의에서 논의한 Figma 링크 (선택) |

### 회의록 목록
- 최신순 정렬
- 회의 유형별 필터
- 월간 캘린더 뷰
- 검색

### 액션 아이템 추적
- 회의록에서 생성된 액션 아이템 → 업무(Task)로 변환 가능
- 미완료 액션 아이템 알림

---

## 4.4 팀 위키

### 설명
팀 내부 지식을 문서화하여 축적하는 공간

### 문서 구조
- 카테고리별 트리 구조
- 예시 카테고리:
  - 온보딩 가이드
  - 코딩 컨벤션
  - 디자인 시스템 가이드
  - 인프라 / 배포 가이드
  - API 문서
  - 트러블슈팅

### 문서 기능
- 마크다운 에디터 (코드 블록, 테이블, 이미지 지원)
- 문서 버전 히스토리
- 최근 수정자 표시
- 문서 내 검색
- Figma 임베드 (디자인 시스템 문서에 Figma 미리보기 삽입)
