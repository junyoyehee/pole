# Phase 2: Figma 허브

## 목표
Figma를 팀 협업의 중심으로 활용합니다. 디자인 파일 탐색, 디자인 리뷰, 디자인-개발 핸드오프를 인트라넷 내에서 처리합니다.

---

## 2.1 디자인 허브 (메인)

### 설명
팀에서 관리하는 모든 Figma 프로젝트를 한 곳에서 관리하는 대시보드

### Figma 프로젝트 등록
| 항목 | 설명 |
|------|------|
| 프로젝트 이름 | 표시 이름 (예: "미르M 플랫폼") |
| Figma 파일 URL | Figma 파일 링크 |
| 담당 디자이너 | 팀원 선택 |
| 담당 개발자 | 팀원 선택 (복수) |
| 상태 | 디자인중 / 리뷰중 / 개발중 / 완료 |
| 설명 | 프로젝트 간단 설명 |

### 프로젝트 카드 뷰
- 카드에 Figma 썸네일 표시 (Figma API 이미지)
- 상태 뱃지
- 담당자 아바타
- 최근 업데이트 시간
- 필터: 상태별, 담당자별

### 프로젝트 상세
- Figma 파일 구조 트리 (기존 Figma Explorer 통합)
- 등록된 디자인 리뷰 목록
- 핸드오프 항목 목록
- 관련 프로젝트 업무 링크

---

## 2.2 Figma 탐색기 (기존 확장)

### 기존 기능 유지
- URL 입력 → 트리 탐색
- 노드 미리보기
- Node ID / MCP 명령어 / URL 복사

### 추가 기능

#### 트리 내 검색
- 노드 이름 검색 (실시간 필터링)
- 검색 결과 하이라이팅
- 타입별 필터 (FRAME, COMPONENT 등)

#### 즐겨찾기
- 자주 사용하는 노드 북마크
- 즐겨찾기 목록 사이드 패널
- 팀원 간 공유 가능

#### 최근 조회 히스토리
- 최근 열어본 Figma 파일 목록
- 파일별 마지막 접근 시간

#### 디자인 토큰 추출
- 선택 노드의 색상, 타이포그래피, 간격 정보 표시
- CSS / Tailwind / Mantine 코드 스니펫 자동 생성
- 복사 버튼

## 2.3 디자인 리뷰

### 설명
디자이너가 올린 디자인에 대해 팀원들이 피드백을 남기는 리뷰 시스템

### 리뷰 등록
| 항목 | 설명 |
|------|------|
| 리뷰 제목 | 리뷰 주제 (예: "로그인 화면 v2 리뷰") |
| Figma 링크 | 리뷰 대상 Figma URL (node-id 포함) |
| 설명 | 리뷰 요청 배경, 확인 포인트 |
| 리뷰어 지정 | 피드백 요청할 팀원 |
| 마감일 | 리뷰 기한 |

### 리뷰 화면
- 좌측: Figma 디자인 이미지 (API로 렌더링)
- 우측: 피드백 스레드
- 피드백 상태: 의견 / 수정요청 / 승인
- 전체 리뷰 상태: 진행중 / 수정중 / 승인완료

### 피드백 작성
- 텍스트 피드백
- 카테고리: UI / UX / 접근성 / 기술적 / 기타
- 우선순위: 높음 / 보통 / 낮음

---

## 2.4 디자인-개발 핸드오프

### 설명
디자인이 확정된 후 개발자에게 전달하는 핸드오프 관리

### 핸드오프 항목
| 항목 | 설명 |
|------|------|
| 컴포넌트 이름 | 구현할 컴포넌트 이름 |
| Figma Node | Figma URL + node-id |
| 담당 개발자 | 구현 담당자 |
| 상태 | 대기 / 진행중 / 구현완료 / 검수완료 |
| 우선순위 | 높음 / 보통 / 낮음 |
| 디자인 스펙 | Figma에서 추출한 스타일 정보 (자동) |
| 비고 | 개발 시 참고사항 |

### 핸드오프 보드
- 칸반 뷰 (대기 → 진행중 → 구현완료 → 검수완료)
- 리스트 뷰 (테이블)
- 필터: 담당자별, 상태별, 프로젝트별

### 핸드오프 상세
- Figma 미리보기 이미지
- 디자인 스펙 (색상, 폰트, 간격 등 자동 추출)
- 코드 스니펫 (Tailwind / CSS / Mantine)
- 댓글 스레드 (디자이너-개발자 소통)
