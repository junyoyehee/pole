# Phase 3: 프로젝트 관리

## 목표
팀의 프로젝트와 업무를 체계적으로 관리합니다. Figma 허브의 핸드오프와 연계하여 디자인→개발 흐름을 추적합니다.

---

## 3.1 프로젝트

### 프로젝트 생성
| 항목 | 설명 |
|------|------|
| 프로젝트 이름 | 예: "미르M 플랫폼 리뉴얼" |
| 설명 | 프로젝트 개요 |
| 시작일 / 마감일 | 프로젝트 기간 |
| PM | 프로젝트 매니저 (팀원 선택) |
| 참여 멤버 | 관련 팀원 (복수) |
| 연결 Figma 프로젝트 | Phase 2의 디자인 허브 프로젝트 연결 |
| 상태 | 준비중 / 진행중 / 보류 / 완료 |
| 우선순위 | 긴급 / 높음 / 보통 / 낮음 |

### 프로젝트 목록
- 카드 뷰 / 리스트 뷰 전환
- 상태별 필터
- 나의 프로젝트 / 전체 프로젝트 탭
- 진행률 표시 (완료 업무 / 전체 업무)

### 프로젝트 상세
- 프로젝트 정보 헤더 (이름, 기간, 상태, PM, 진행률)
- 업무 보드 (칸반)
- 업무 리스트 (테이블)
- 타임라인 (간트 차트 간소화)
- 연결된 Figma 디자인 목록
- 활동 로그

---

## 3.2 업무 (Task)

### 업무 생성
| 항목 | 설명 |
|------|------|
| 제목 | 업무 제목 |
| 설명 | 마크다운 지원 상세 설명 |
| 담당자 | 팀원 선택 (복수) |
| 상태 | TODO / IN_PROGRESS / IN_REVIEW / DONE |
| 우선순위 | 긴급 / 높음 / 보통 / 낮음 |
| 마감일 | 업무 기한 |
| 라벨 | Frontend / Backend / Design / Bug / Feature 등 |
| Figma 링크 | 관련 Figma node 연결 (선택) |
| 하위 업무 | 체크리스트 형태 |

### 업무 보드 (칸반)
```
┌──────────┬──────────┬──────────┬──────────┐
│   TODO   │ 진행중    │  리뷰    │   완료   │
├──────────┼──────────┼──────────┼──────────┤
│ [카드]    │ [카드]   │ [카드]    │ [카드]   │
│ [카드]    │ [카드]   │          │ [카드]   │
│ [카드]    │          │          │ [카드]   │
└──────────┴──────────┴──────────┴──────────┘
```
- 드래그앤드롭으로 상태 변경
- 카드에 표시: 제목, 담당자 아바타, 우선순위, 라벨, 마감일
- 필터: 담당자, 라벨, 우선순위

### 업무 상세
- 상세 설명 (마크다운 렌더링)
- 하위 업무 체크리스트
- Figma 미리보기 (연결된 경우)
- 댓글 스레드
- 활동 로그 (상태 변경, 담당자 변경 등)
- 파일 첨부

## 3.3 내 업무

### 설명
로그인한 사용자에게 할당된 업무를 모아보는 개인 뷰

### 표시 내용
- 오늘 마감 업무
- 진행중인 업무
- 리뷰 요청된 업무 (디자인 리뷰 + 업무 리뷰)
- 최근 완료 업무
- 프로젝트별 그룹핑 가능

---

## 3.4 타임라인

### 설명
프로젝트의 전체 일정을 시각적으로 표시

### 기능
- 간트 차트 형태 (간소화)
- 업무별 기간 바 표시
- 마일스톤 표시
- 오늘 날짜 기준선
- 주간 / 월간 뷰 전환
