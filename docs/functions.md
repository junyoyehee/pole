# Figma Explorer - 기능 상세 명세

## 1. URL 파싱

### 설명
사용자가 입력한 Figma URL에서 `fileKey`와 `nodeId`를 자동으로 추출합니다.

### 지원 URL 형식
```
https://www.figma.com/design/<fileKey>/<fileName>?node-id=<nodeId>
https://www.figma.com/file/<fileKey>/<fileName>?node-id=<nodeId>
```

### 동작 방식
1. 정규식으로 URL에서 `fileKey` 추출: `/figma\.com\/(file|design)\/([a-zA-Z0-9]+)/`
2. URL 쿼리 파라미터에서 `node-id` 값 추출
3. `node-id`가 없으면 파일 전체를 조회 대상으로 설정

### 입출력 예시
| 입력 URL | fileKey | nodeId |
|----------|---------|--------|
| `https://figma.com/design/ABC123/MyFile?node-id=42-15` | `ABC123` | `42-15` |
| `https://figma.com/file/XYZ789/MyFile` | `XYZ789` | (빈 문자열) |
| `잘못된 URL` | (빈 문자열) | (빈 문자열) |

### 에러 처리
- `fileKey`를 추출할 수 없는 경우 "올바른 Figma URL을 입력해주세요." 에러 메시지 표시

## 2. 파일 구조 조회

### 설명
Figma REST API를 호출하여 파일 또는 특정 노드의 트리 구조를 가져옵니다.

### API 흐름
```
[사용자 입력] → [프론트엔드] → GET /api/figma → [Figma REST API] → [응답 반환]
```

### 두 가지 조회 모드

#### 모드 A: 파일 전체 조회 (nodeId 없음)
- Figma API 엔드포인트: `GET /v1/files/:fileKey?depth=:depth`
- 파일의 모든 페이지(CANVAS)와 하위 노드를 반환
- 응답 구조: `data.document.children` 배열을 트리 루트로 사용

#### 모드 B: 특정 노드 조회 (nodeId 있음)
- Figma API 엔드포인트: `GET /v1/files/:fileKey/nodes?ids=:nodeId&depth=:depth`
- 지정된 노드와 하위 구조만 반환
- 응답 구조: `data.nodes[nodeId].document`를 트리 루트로 사용

### 깊이(Depth) 파라미터
조회할 트리의 깊이를 제한합니다. Figma API의 `depth` 파라미터에 전달됩니다.

| 값 | 동작 | 권장 상황 |
|----|------|----------|
| 1 | 직계 자식만 | 페이지 목록 확인 |
| 2 | 2단계까지 (기본값) | 페이지 + 최상위 프레임 |
| 3 | 3단계까지 | 프레임 내부 컴포넌트 확인 |
| 4 | 4단계까지 | 컴포넌트 내부 레이어 확인 |
| 미설정 | 전체 | 모든 하위 노드 (대형 파일 주의) |

### nodeId 형식 변환
- 프론트엔드/URL에서는 하이픈 형식: `42-15`
- Figma API에서는 콜론 형식: `42:15`
- API 라우트에서 자동 변환: `.replace("-", ":")`

## 3. 트리 뷰 렌더링

### 설명
조회된 노드 데이터를 재귀적 트리 구조로 시각화합니다.

### 트리 노드 구성 요소
각 트리 노드는 한 줄로 표시되며 다음 요소를 포함합니다:

```
[▾/▸] [TYPE 뱃지] [노드 이름]                    [node-id 코드]
```

| 요소 | 설명 |
|------|------|
| 펼침/접힘 화살표 | 하위 노드가 있는 경우에만 표시. 클릭으로 토글 |
| TYPE 뱃지 | 노드 타입을 컬러 뱃지로 표시 |
| 노드 이름 | Figma에서 설정한 레이어 이름. 길면 말줄임 처리 |
| node-id 코드 | 클릭 시 클립보드에 복사. 복사 성공 시 색상 변경(teal) |

### 들여쓰기
- 깊이(depth)에 따라 좌측 패딩 증가: `depth * 20 + 8` px
- 시각적으로 부모-자식 관계를 표현

### 초기 펼침 상태
- `depth < 1`인 노드만 자동 펼침 (최상위 레벨)
- 나머지는 접힌 상태로 시작

### 선택 상태
- 선택된 노드는 파란색 배경(`blue-0`) + 파란색 테두리(`blue-3`)로 강조
- 선택 시 우측 상세 패널에 정보 표시

## 4. 노드 타입 분류

### 설명
Figma의 노드 타입에 따라 서로 다른 색상의 뱃지를 표시하여 구분합니다.

### 타입별 색상 매핑

| 타입 | 색상 | Mantine 컬러 | Figma에서의 의미 |
|------|------|-------------|-----------------|
| CANVAS | 보라색 | `violet` | 페이지. 파일의 최상위 단위 |
| FRAME | 파란색 | `blue` | 프레임/아트보드. 레이아웃 컨테이너 |
| COMPONENT | 초록색 | `green` | 재사용 가능한 마스터 컴포넌트 |
| COMPONENT_SET | 틸색 | `teal` | Variants를 포함하는 컴포넌트 세트 |
| INSTANCE | 시안색 | `cyan` | 컴포넌트의 인스턴스 (복제본) |
| GROUP | 노란색 | `yellow` | 단순 그룹 |
| TEXT | 주황색 | `orange` | 텍스트 레이어 |
| RECTANGLE | 회색 | `gray` | 사각형 도형 |
| VECTOR | 핑크색 | `pink` | 벡터 도형 (아이콘 등) |
| SECTION | 인디고색 | `indigo` | 섹션 (Figma의 정리 도구) |
| 기타 | 회색 | `gray` | 위에 해당하지 않는 타입 |

## 5. 노드 미리보기

### 설명
트리에서 노드를 선택하면 Figma API를 통해 해당 노드의 PNG 이미지를 렌더링하여 표시합니다.

### API 흐름
```
[노드 클릭] → GET /api/figma/images?fileKey=...&nodeId=...
            → Figma API: GET /v1/images/:fileKey?ids=:nodeId&format=png&scale=1
            → 이미지 URL 반환 → <img> 태그로 표시
```

### 이미지 옵션
| 항목 | 값 |
|------|-----|
| 포맷 | PNG |
| 스케일 | 1x |
| 표시 방식 | `object-fit: contain` (비율 유지) |

### 상태별 UI

| 상태 | 표시 |
|------|------|
| 로딩 중 | Skeleton 애니메이션 (높이 200px) |
| 이미지 로드 성공 | 회색 배경 위에 PNG 이미지 표시 |
| 이미지 로드 실패 | "미리보기를 불러올 수 없습니다" 안내 텍스트 |

### 참고사항
- Figma API가 반환하는 이미지 URL은 임시 URL (유효기간 있음)
- CANVAS(페이지) 타입은 미리보기가 생성되지 않을 수 있음
- 매우 큰 프레임은 렌더링에 시간이 걸릴 수 있음

## 6. 클립보드 복사

### 설명
선택한 노드의 정보를 3가지 형태로 클립보드에 복사할 수 있습니다.

### 6-1. Node ID 복사

| 항목 | 내용 |
|------|------|
| 복사 형식 | `42-15` |
| 용도 | Figma URL 조합, MCP 명령어 직접 작성 |
| 복사 위치 | 상세 패널의 "Node ID 복사" 버튼 또는 트리 노드 우측의 코드 클릭 |

### 6-2. MCP 명령어 복사

| 항목 | 내용 |
|------|------|
| 복사 형식 | `get_figma_data(fileKey="ABC123", nodeId="42-15")` |
| 용도 | Claude Code 대화창에 붙여넣어 Figma MCP 도구로 즉시 조회 |
| 복사 위치 | 상세 패널의 "MCP 명령어" 버튼 |

### 6-3. Figma URL 복사

| 항목 | 내용 |
|------|------|
| 복사 형식 | `https://www.figma.com/design/ABC123/?node-id=42-15` |
| 용도 | Figma에서 해당 노드로 바로 이동, Claude Code에 URL 전달 |
| 복사 위치 | 상세 패널의 "Figma URL" 버튼 |

### 복사 피드백
- 복사 전: 각 버튼 고유 색상 (회색, 파랑, 보라)
- 복사 후: 버튼 색상이 `teal`로 변경, 텍스트가 "복사됨!"으로 전환
- Mantine `CopyButton` 컴포넌트가 자동으로 상태 관리

## 7. 노드 상세 정보 패널

### 설명
트리에서 선택한 노드의 상세 정보를 우측 패널에 표시합니다.

### 패널 구성

```
┌─────────────────────────┐
│ 노드 이름               │
│ [TYPE 뱃지] node-id     │
├─────────────────────────┤
│ [Node ID] [MCP] [URL]   │
├─────────────────────────┤
│ 미리보기                 │
│ ┌─────────────────────┐ │
│ │                     │ │
│ │    PNG 이미지        │ │
│ │                     │ │
│ └─────────────────────┘ │
├─────────────────────────┤
│ 정보                     │
│ • 하위 요소 N개          │
└─────────────────────────┘
```

### 표시 항목

| 영역 | 내용 |
|------|------|
| 헤더 | 노드 이름, 타입 뱃지, 원본 ID (콜론 형식) |
| 액션 버튼 | Node ID / MCP 명령어 / Figma URL 복사 |
| 미리보기 | PNG 이미지 또는 로딩/에러 상태 |
| 정보 | 하위 요소 개수 (children이 있는 경우만) |

### 미선택 상태
- 노드가 선택되지 않은 경우 "노드를 선택하면 상세 정보가 표시됩니다" 안내 표시

## 8. 에러 처리

### 설명
API 호출 실패 및 잘못된 입력에 대한 에러 처리입니다.

### 에러 유형별 처리

| 에러 상황 | 표시 방식 | 메시지 |
|----------|----------|--------|
| 잘못된 URL 입력 | Alert (빨간색) | "올바른 Figma URL을 입력해주세요." |
| API 키 미설정 | Alert (빨간색) | "FIGMA_API_KEY is not configured in .env.local" |
| Figma API 오류 | Alert (빨간색) | "Figma API error: {상태코드}" |
| 네트워크 오류 | Alert (빨간색) | "Figma API 요청에 실패했습니다." |
| 이미지 로드 실패 | 패널 내 안내 | "미리보기를 불러올 수 없습니다" |

### Alert 동작
- 닫기 버튼(X)으로 에러 메시지 숨기기 가능
- 새로운 조회 시 이전 에러 자동 초기화

---

## 9. 로딩 상태

### 설명
API 요청 중 사용자에게 진행 상태를 시각적으로 알려줍니다.

### 로딩 표시

| 상황 | UI |
|------|-----|
| 파일 구조 조회 중 | 화면 중앙에 Loader 스피너 + "Figma 파일을 불러오는 중..." 텍스트 |
| 노드 이미지 로딩 중 | 상세 패널 미리보기 영역에 Skeleton 애니메이션 |
| 탐색 버튼 | 로딩 중 비활성화 + 스피너 표시 (Mantine `loading` prop) |

## 10. 레이아웃 구조

### 설명
Mantine AppShell을 기반으로 한 2패널 레이아웃입니다.

### 전체 레이아웃

```
┌──────────────────────────────────────────────┐
│ [F] Figma Explorer              파일명 | 키  │
│ [URL 입력창              ] [깊이 ▾] [탐색]   │
├────────────────────────────┬─────────────────┤
│                            │                 │
│         노드 트리           │   상세 패널      │
│        (flex: 1)           │   (420px 고정)   │
│                            │                 │
│     ScrollArea             │   ScrollArea    │
│                            │                 │
└────────────────────────────┴─────────────────┘
```

### 영역별 설명

| 영역 | 높이/너비 | 스크롤 | 설명 |
|------|----------|--------|------|
| 헤더 | 130px 고정 | 없음 | URL 입력, 깊이 선택, 탐색 버튼, 파일 정보 |
| 트리 패널 | `calc(100vh - 170px)` | 세로 스크롤 | 노드 트리. 남은 너비 전체 사용 |
| 상세 패널 | `calc(100vh - 170px)` | 세로 스크롤 | 선택 노드 정보. 420px 고정 너비 |
